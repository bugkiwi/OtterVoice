Project guide · Architecture

基础能力负责实时，业务只需要定义对话。

01 · INPUT

Runtime

Web Audio 或 Expo PCM 负责采集、音量和播放。

02 · SESSION

Core

状态机组织 turn、并发、打断、取消和错误恢复。

03 · INTELLIGENCE

Providers

Audio LLM，或可替换的 ASR → LLM → TTS 级联通路。

04 · OUTPUT

Experience

字幕、首音频延迟、连续播放与自然插话。

@ottervoice/core

跨平台 VoiceSession、事件和全双工策略。

@ottervoice/runtime-web

MediaRecorder、Web Audio VAD 与流式 PCM 播放。

@ottervoice/runtime-react-native

Expo 原生 PCM 输入、AudioPlaylist 输出与文件清理。

@ottervoice/provider-openrouter

Audio LLM、ASR、LLM 和 TTS 的统一 OpenRouter 接入。

@ottervoice/provider-utils

SSE、HTTP 错误和 provider 公共能力。

examples/

可直接运行的 Web、Expo 和 Node 集成样板。

密钥不进入浏览器或 App。

示例统一请求 ottervoice.vercel.app 的同源代理，由服务端注入 OpenRouter 凭据；生产项目可以替换为自己的 token broker。
