01

快速开始Quick start

先用 Mock Runtime 和 Mock Provider 验证 UI 与会话逻辑，不需要 API Key。生产环境只需替换 runtime 和 providers。

Start with mock runtimes and providers to validate the UI and session loop without API keys. Production integration only replaces runtime and providers.

Install
bun add @ottervoice/core
# or
npm install @ottervoice/core
TypeScript
import {
  createMockASR,
  createMockLLM,
  createMockRuntime,
  createMockTTS,
  createVoiceSession,
} from '@ottervoice/core';

const runtime = createMockRuntime();
const session = createVoiceSession({
  mode: 'half_duplex',
  runtime,
  providers: {
    asr: createMockASR({ transcripts: ['Hello Otter'] }),
    llm: createMockLLM({ reply: () => 'Hello! How can I help?' }),
    tts: createMockTTS(),
  },
});

session.on('asr_partial', ({ text }) => console.log('user:', text));
session.on('assistant_text_delta', ({ text }) => console.log('assistant:', text));

await session.start();
先完成事件接入，再连接真实服务。

Mock 与真实 Provider 共享同一个事件协议，UI 无需写两套逻辑。

Wire events before connecting a live service.

Mocks and real providers share the same event contract, so the UI needs only one implementation.

02

架构Architecture

OtterVoice 是语音会话编排 SDK，不是模型服务或固定网关。核心层负责状态、轮次、并发、打断、错误与用量；平台音频和模型连接通过接口注入。

OtterVoice is a voice-session orchestration SDK, not a model service or fixed gateway. The core owns state, turns, concurrency, interruption, errors, and usage; platform audio and model connections are injected.

两种流水线

  • asr_llm_tts：ASR 产生文本，LLM 流式生成文本，TTS 播放最终回复。
  • audio_llm：完整用户音频进入 Audio LLM；ASR 并行生成字幕；回复文本和 PCM 可边生成边输出。

Two pipelines

  • asr_llm_tts: ASR produces text, the LLM streams text, and TTS plays the completed reply.
  • audio_llm: completed user audio enters an Audio LLM while ASR captions run in parallel; reply text and PCM can stream immediately.
03

会话配置Session configuration

在应用组合根创建 Session。UI 只依赖 Session 事件，不应读取 Provider URL 或模型凭证。

Create the session at the application composition root. UI code should depend on session events, not provider URLs or model credentials.

VoiceSession
const session = createVoiceSession({
  mode: 'full_duplex',
  pipeline: 'asr_llm_tts',
  runtime,
  providers: { asr, llm, tts },
  language: 'zh-CN',
  turnDetection: {
    strategy: 'volume',
    minSpeechMs: 180,
    silenceTimeoutMs: 450,
    volumeThreshold: 0.025,
  },
  interruptionDetection: {
    minSpeechMs: 160,
    silenceTimeoutMs: 350,
    volumeThreshold: 0.018,
  },
  policy: {
    autoStartListening: true,
    allowInterruption: true,
    interruptionTailIgnoreMs: 200,
    falseInterruptionSilenceMs: 400,
    falseInterruptionTimeoutMs: 1_200,
  },
});

await session.start();
await session.finish();
await session.dispose();

示例 VAD 数值适合作为低延迟起点，不是所有设备的固定值。应使用目标设备、扬声器音量和实际噪声样本调参。

The example VAD values are low-latency starting points, not universal defaults. Tune with target devices, speaker volume, and representative noise.

04

实时文本事件Streaming text events

所有增量与最终事件都带有稳定的 turnId。页面必须原位更新一条消息，不能把每个 delta 追加成新消息。

Every incremental and final event has a stable turnId. Upsert one message in place; never append every delta as a new row.

UI event contract
session.on('asr_partial', ({ turnId, text }) => {
  upsert({ id: turnId, role: 'user', text, live: true });
});

session.on('asr_final', ({ turnId, text }) => {
  upsert({ id: turnId, role: 'user', text, live: false });
});

session.on('assistant_text_delta', ({ turnId, text }) => {
  upsert({ id: turnId, role: 'assistant', text, live: true });
});

session.on('assistant_text', ({ turnId, text }) => {
  upsert({ id: turnId, role: 'assistant', text, live: false });
});

asr_final用户轮次确认User turn confirmed覆盖临时文本Replace provisional text
assistant_text_delta模型返回新文本Model emits text原位更新助手消息Update the assistant row
assistant_text回复确认Reply confirmed最终文本收口Commit final text

text 是累计值。

assistant_text_delta.delta 只是本次新增片段；渲染时优先使用累计 text，最终事件可修正标点和空格。

text is accumulated.

assistant_text_delta.delta contains only the new fragment. Render accumulated text; the final event may normalize punctuation and spacing.

05

Provider adapters

Provider 是扩展边界。替换时只在组合根创建另一实现，Session、Runtime 与 UI 不变。自定义 ASR 必须准确声明 capabilities.streaming，核心据此选择 live 分片或完整轮次音频。

Providers are the extension boundary. Replace an implementation at the composition root without changing the session, runtime, or UI. Custom ASR adapters must declare capabilities.streaming accurately.

能力Capability仓库适配器Included adapters传输Transport
Streaming ASRDeepgram · ElevenLabsWebSocket partial / final
Batch / rolling ASROpenRouter （可选 Demo）(optional demo)HTTP snapshot
Streaming LLMOpenRouter （可选）(optional)SSE text delta
TTSAzure Speech · OpenRouterHTTP audio
Audio LLMOpenRouter （可选 Demo）(optional demo)SSE transcript + PCM
OpenRouter 不是 SDK 的固定线上通路。

它只是仓库提供的一个可选适配器和 Demo 当前选择。生产应用可以使用地区服务、自托管模型或任意实现同一接口的 Provider。

OpenRouter is not a fixed OtterVoice route.

It is one optional adapter and the current demo choice. Production applications can use regional, self-hosted, or any interface-compatible provider.

06

Web 接入Web runtime

Web Runtime 使用 getUserMedia、MediaRecorder 和 Web Audio。Demo 每 100 ms 产生 WebM/Opus 分片，每 500 ms 请求一次滚动 partial；原生流式 ASR 可直接消费这些分片。

The Web runtime uses getUserMedia, MediaRecorder, and Web Audio. The demo emits WebM/Opus every 100 ms and requests rolling partials every 500 ms; native streaming ASR can consume the same chunks.

Web runtime
import { createWebRuntime } from '@ottervoice/runtime-web';

const runtime = createWebRuntime({
  mimeType: 'audio/webm;codecs=opus',
  timesliceMs: 100,
  volumePollMs: 50,
});

浏览器要求

  • 页面必须运行在 HTTPS 或 localhost。
  • 通过用户手势请求麦克风权限并尝试解锁播放。
  • autoplay 解锁失败不应阻断输入与 ASR。
  • 结束页面时调用 dispose() 释放 track、AudioContext 和播放器。

Browser requirements

  • Serve over HTTPS or localhost.
  • Request microphone access and attempt playback unlock from a user gesture.
  • An autoplay-unlock rejection must not block input or ASR.
  • Call dispose() to release tracks, AudioContext, and playback.

Expo 接入Expo runtime

React Native Runtime 通过小型接口注入 Expo 能力，避免 core 直接依赖 Expo 包。推荐输入为 16 kHz、mono、PCM16。

The React Native runtime accepts small injected Expo interfaces so the core never depends directly on Expo packages. Use 16 kHz mono PCM16 input.

08

延迟优化Latency

可感知延迟不是一个请求耗时，而是静音判定、上传、模型首 token / PCM 和播放器排队的总和。

Perceived latency is not one request duration. It is endpointing, upload, model time-to-first-token / PCM, and player queueing combined.

  1. 使用实时音频分片和支持 partial 的 ASR，让输入字幕在说话时出现。Use live audio chunks and partial ASR so captions appear while the user speaks.
  2. 在噪声测试后把 silenceTimeoutMs 调整到约 350–600 ms。Tune silenceTimeoutMs around 350–600 ms after noise testing.
  3. 实现 LLMProvider.stream() 或 Audio LLM 的文本 / PCM 回调。Implement LLMProvider.stream() or Audio LLM transcript / PCM callbacks.
  4. 复用连接和播放器，限制回复长度与输出 token。Reuse connections and playback resources; limit reply length and output tokens.
  5. 用 user_audio_end → assistant_audio_start 衡量真实首音频延迟。Measure real first-audio latency with user_audio_end → assistant_audio_start.
滚动 partial 是兼容方案。

它会重复上传累计音频，适合 Demo 或批量 ASR。高并发生产环境优先使用原生流式 ASR。

Rolling partials are a compatibility path.

They repeatedly upload accumulated audio. Prefer native streaming ASR for high-scale production.

09

安全与网关Security and gateways

浏览器、App 二进制和 EXPO_PUBLIC_* 变量都不能保存长期 Provider Key。客户端只连接你控制的服务端网关或短期令牌 broker。

Never ship long-lived provider keys in browser bundles, app binaries, or EXPO_PUBLIC_* variables. Clients connect only to a controlled gateway or short-lived token broker.

Token broker request
POST /api/voice/token
Content-Type: application/json

{
  "provider": "deepgram",
  "purpose": "asr",
  "sessionId": "optional"
}
Token broker response
{
  "token": "short-lived-token",
  "url": "wss://optional-signed-endpoint",
  "expiresAt": 1784000000000
}

网关必须执行用户鉴权、Provider / 模型白名单、配额、速率限制、请求大小限制、超时和审计 ID。默认不要记录原始音频、完整 transcript 或上游凭证。

The gateway must enforce authentication, provider/model allowlists, quotas, rate limits, payload limits, timeouts, and audit IDs. Do not log raw audio, full transcripts, or upstream credentials by default.

错误、测试与上线Errors, tests, and release

Observability
session.on('error', ({ code, message, provider, retryable }) => {
  logger.error({ code, provider, retryable }, message);
});

session.on('usage', (usage) => {
  metrics.record('voice_session_ms', usage.sessionDurationMs);
  metrics.record('asr_audio_ms', usage.asrAudioMs);
});

常见错误码

microphone_unavailable、asr_connection_failed、asr_timeout、llm_failed、tts_failed、audio_playback_failed、provider_rate_limited、provider_quota_exceeded、unsupported_runtime、invalid_state。

上线检查

  • 麦克风拒绝、空转录、断网、429 与 5xx。
  • 中英文 partial / final 是否按 turnId 原位更新。
  • 噪声、短暂停顿、扬声器回声、真实插话和误打断恢复。
  • iOS / Android 音频焦点、后台和来电中断。
  • 首字幕、首文本、首音频和总轮次延迟。
  • 客户端包与日志中没有长期 Provider Key。

Common error codes

microphone_unavailable, asr_connection_failed, asr_timeout, llm_failed, tts_failed, audio_playback_failed, provider_rate_limited, provider_quota_exceeded, unsupported_runtime, and invalid_state.

Release checklist

  • Permission denial, empty transcripts, disconnects, 429, and 5xx.
  • Incremental bilingual UI upserts by turnId.
  • Noise, short pauses, speaker echo, real barge-in, and false-interruption recovery.
  • iOS / Android audio focus, backgrounding, and phone interruptions.
  • First-caption, first-text, first-audio, and total-turn latency.
  • No long-lived provider keys in client artifacts or logs.
